# 최적화 분석

제공해주신 코드는 0-1 BFS 알고리즘을 사용하고 있어, 이미 최단 경로를 찾는 데 있어 매우 효율적인 방법입니다. `N-1`, `N+1` 이동에는 비용 1이 들고, `2*N` 이동에는 비용 0이 드는 특성 때문에 0-1 BFS가 최적입니다.

하지만 몇 가지 측면에서 최적화 및 개선이 가능합니다.

---

### 분석 및 개선점

**1. 시간 복잡도 개선 방법**
*   **현재 코드의 시간 복잡도:** O(V + E)
    *   V (정점의 수): 최대 위치 (`MAX_POSITION` 까지)
    *   E (간선의 수): 각 정점에서 3개의 간선 (N-1, N+1, 2*N)
    *   0-1 BFS는 다익스트라 알고리즘의 특수한 경우로, 힙 대신 덱(Deque)을 사용하여 0-비용 간선은 덱의 앞에, 1-비용 간선은 덱의 뒤에 추가함으로써 O(V+E)의 시간 복잡도를 달성합니다. 이는 해당 문제에 대한 최적의 시간 복도입니다.
*   **개선 방법:** 추가적인 점근적 시간 복도 개선은 어렵습니다. 현재 로직 자체가 이미 최적입니다.
    *   다만, BFS 내부의 `for` 루프를 명확한 `if` 블록으로 분리하여 조건문 평가 오버헤드를 줄이고 가독성을 높일 수 있습니다. 이는 미미한 성능 향상에 불과하지만, 코드의 의도를 더 명확하게 합니다.

**2. 공간 복잡도 개선 방법**
*   **현재 코드의 공간 복잡도:** O(V)
    *   `dist` 배열: `100002` 크기
    *   `Deque`: 최악의 경우 모든 정점을 저장할 수 있으므로 `100002` 크기
*   **개선 방법:**
    *   **`dist` 배열의 크기 조정:** 현재 `dist` 배열의 크기는 `100002`로 설정되어 있습니다. 하지만 `K`가 `100000`일 때, `N * 2` 연산으로 인해 `200000`까지 도달할 수 있습니다. 예를 들어, `N=50000`, `K=100000`인 경우 `2*N = 100000`은 문제가 없지만, `N=70000`, `K=100000`일 때 `2*N = 140000`까지 탐색해야 할 수도 있습니다. 따라서 `dist` 배열의 최대 크기는 `2 * K_MAX`까지 고려해야 합니다. (여기서 `K_MAX = 100000`).
        *   **변경 예시:** `int[] dist = new int[200001];` (인덱스 0부터 200000까지 사용)
    *   이것은 "개선"이라기보다는 "정확성 확보"에 가깝습니다. 공간 복잡도 자체는 O(V)로 동일하지만, V의 상한을 정확히 설정하여 올바른 동작을 보장합니다.

**3. 가독성 개선**
*   **상수 사용:** `100001`, `100002`와 같은 "매직 넘버" 대신 명확한 상수를 사용하는 것이 좋습니다.
    *   **변경 예시:** `private static final int MAX_POSITION = 200000;`
*   **BFS 로직 명확화:** `for (int i = 0; i < 3; i++)`와 삼항 연산자를 이용한 `next` 계산은 다소 복잡해 보일 수 있습니다. 각 이동 유형(`-1`, `+1`, `*2`)을 별도의 `if` 블록으로 처리하여 가독성을 높일 수 있습니다.
    *   **변경 예시:** `-1`, `+1`, `*2` 연산을 각각의 `if` 블록으로 분리.
*   **`dist` 배열 접근:** `bfs` 메서드에서 `dist` 배열을 매개변수로 전달하는 대신, 클래스의 정적(static) 멤버로 선언하여 `bfs` 메서드 시그니처를 간결하게 만들 수 있습니다.
    *   **변경 예시:** `private static int[] dist;`
*   **Scanner 닫기:** `Scanner` 객체 사용 후 `sc.close()`를 호출하여 리소스를 해제하는 것이 좋습니다.

### 4. 최종 최적화된 코드

```java
import java.util.ArrayDeque;
import java.util.Arrays;
import java.util.Deque;
import java.util.Scanner;

public class BOJ_13549_Optimized {

    // 문제의 최대 위치 N, K <= 100,000. 순간이동(2*N)을 고려하면 200,000까지 도달할 수 있음.
    private static final int MAX_POSITION = 200000;
    // dist 배열을 클래스의 정적 멤버로 선언하여 bfs 메서드의 매개변수를 줄임
    private static int[] dist; 

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int N = sc.nextInt();
        int K = sc.nextInt();
        sc.close(); // Scanner 리소스 해제

        // dist 배열을 MAX_POSITION + 1 크기로 초기화 (0부터 MAX_POSITION까지 인덱스 사용)
        dist = new int[MAX_POSITION + 1];
        Arrays.fill(dist, Integer.MAX_VALUE); // 모든 거리를 무한대로 초기화

        int result = bfs(N, K); // BFS 시작
        System.out.println(result);
    }

    // BFS 메서드: Deque와 dist 배열은 내부에서 관리하거나 정적 멤버 사용
    public static int bfs(int start, int end) {
        Deque<Integer> q = new ArrayDeque<>();

        dist[start] = 0; // 시작 지점의 시간은 0
        q.offer(start); // 시작 지점을 큐에 추가

        while (!q.isEmpty()) {
            int current = q.poll(); // 현재 위치를 큐에서 꺼냄

            // 목표 지점에 도달했으면 시간 반환
            if (current == end) {
                return dist[current];
            }

            // 1. 순간이동 (2 * current): 비용 0
            int nextMultiply2 = current * 2;
            // 다음 위치가 유효 범위 내이고, 현재 경로를 통해 가는 것이 더 빠르다면
            if (nextMultiply2 <= MAX_POSITION && dist[current] < dist[nextMultiply2]) {
                dist[nextMultiply2] = dist[current]; // 비용 0이므로 현재 시간 그대로
                q.offerFirst(nextMultiply2); // 비용 0이므로 덱의 맨 앞에 추가 (우선순위 높음)
            }

            // 2. 걷기 (current - 1): 비용 1
            int nextMinus1 = current - 1;
            // 다음 위치가 유효 범위 내이고, 현재 경로를 통해 가는 것이 더 빠르다면
            if (nextMinus1 >= 0 && dist[current] + 1 < dist[nextMinus1]) {
                dist[nextMinus1] = dist[current] + 1; // 비용 1 증가
                q.offerLast(nextMinus1); // 비용 1이므로 덱의 맨 뒤에 추가
            }

            // 3. 걷기 (current + 1): 비용 1
            int nextPlus1 = current + 1;
            // 다음 위치가 유효 범위 내이고, 현재 경로를 통해 가는 것이 더 빠르다면
            if (nextPlus1 <= MAX_POSITION && dist[current] + 1 < dist[nextPlus1]) {
                dist[nextPlus1] = dist[current] + 1; // 비용 1 증가
                q.offerLast(nextPlus1); // 비용 1이므로 덱의 맨 뒤에 추가
            }
        }
        return -1; // K에 도달할 수 없는 경우 (문제 조건상 K는 항상 도달 가능)
    }
}
```

### 5. 왜 이 방법이 최선인지 설명

1.  **0-1 BFS의 적합성 및 최적 시간 복잡도:**
    *   이 문제는 간선 가중치가 0 또는 1로만 구성된 그래프에서 최단 경로를 찾는 문제입니다. 이러한 특수한 경우에 `ArrayDeque`를 사용하는 0-1 BFS는 다익스트라 알고리즘(힙 사용)보다 더 효율적입니다.
    *   비용 0인 간선으로 연결된 노드는 덱의 앞에 추가하여 다음에 즉시 처리되도록 하고, 비용 1인 간선으로 연결된 노드는 덱의 뒤에 추가하여 나중에 처리되도록 합니다. 이 방식은 방문된 노드가 최단 거리를 가진다는 것을 보장하며, 각 노드를 한 번만 방문(혹은 최단 거리가 갱신될 때 한 번 더)하므로 시간 복잡도는 $O(V+E)$로 그래프 탐색 알고리즘의 이론적 최적치에 도달합니다. $V$는 최대 위치, $E$는 최대 간선 수입니다.

2.  **공간 복잡도의 정확성 확보:**
    *   `dist` 배열의 크기를 `MAX_POSITION + 1` (예: 200001)로 설정하여, `N`과 `K`의 범위(최대 100,000)를 고려했을 때 순간이동(`2 * N`) 연산으로 도달할 수 있는 최대 위치(200,000)까지 모든 노드를 커버할 수 있도록 했습니다. 이는 알고리즘의 정확성을 보장하는 데 필수적이며, 문제의 제약 조건 내에서 필요한 최소한의 공간을 사용합니다. 공간 복잡도는 $O(V)$로 최적입니다.

3.  **가독성 및 유지보수성 향상:**
    *   **명확한 상수 사용:** `MAX_POSITION`과 같은 상수를 도입하여 코드 내의 "매직 넘버"를 제거했습니다. 이는 코드의 의미를 명확히 하고, 향후 제약 조건이 변경될 경우 수정이 용이하게 합니다.
    *   **BFS 로직 분리:** `for` 루프와 삼항 연산자 대신, 각 이동 연산(`-1`, `+1`, `*2`)을 별도의 `if` 블록으로 분리했습니다. 이는 각 연산의 목적과 비용이 명확히 드러나게 하여 코드 가독성을 크게 높였습니다. 또한, `offerFirst`와 `offerLast`를 직접 사용하여 0-1 BFS의 핵심 로직을 더욱 명확하게 보여줍니다.
    *   **정적 멤버 활용:** `dist` 배열을 정적 멤버로 선언하여 `bfs` 메서드의 매개변수를 줄였습니다. 이는 특정 알고리즘의 전역 상태를 나타낼 때 유용한 패턴입니다.
    *   **리소스 관리:** `Scanner` 객체를 사용 후 `close()`를 호출하여 시스템 리소스 누수를 방지했습니다.

이러한 개선 사항들은 알고리즘의 근본적인 효율성을 해치지 않으면서 코드의 견고성, 정확성, 가독성을 향상시켜, 주어진 문제에 대해 "최선"에 가까운 해결책을 제시합니다.
