# 최적화 분석

주어진 코드는 백준 14503번 "로봇 청소기" 문제를 해결하는 자바 코드입니다. 이 코드는 문제의 규칙을 잘 따르고 있지만, 중복된 로직과 가독성 측면에서 개선될 여지가 있습니다.

## 1. 시간 복잡도 개선 방법

**결론: 근본적인 시간 복잡도는 `O(N*M)`으로, 이보다 더 효율적인 방법은 없습니다.**

*   **현재 코드의 시간 복잡도:** `while` 루프는 로봇이 움직임을 멈출 때까지 실행됩니다. 각 `while` 루프 내에서는 다음과 같은 작업이 수행됩니다.
    *   현재 칸 청소: `O(1)`
    *   주변 4칸 탐색: `O(4)` (상수 시간)
    *   후진 또는 좌회전/전진: `O(1)`
*   로봇이 청소하는 칸의 수는 최대 `N*M`개입니다. 로봇이 청소하지 않은 칸 주변을 맴돌거나 후진하는 경우도 있지만, 특정 상황 (예: 모든 주변 칸이 청소되었거나 벽인 경우)에만 발생하며, 결국은 청소할 새 칸을 찾거나 멈추게 됩니다.
*   따라서, 최악의 경우에도 모든 칸을 방문하고 각 칸에서 상수 시간 작업을 수행하므로, 전체 시간 복잡도는 `O(N*M)`에 비례합니다.
*   **개선 방법:** 이 문제는 시뮬레이션 문제이기 때문에, 로봇의 움직임을 그대로 흉내 내는 것 외에 더 나은 접근 방식은 없습니다. 따라서 시간 복잡도 자체를 `O(N*M)`보다 더 낮출 수는 없습니다. 최적화는 주로 **상수 시간 계수**를 줄이고 코드의 **가독성**을 높이는 데 중점을 둡니다. 중복된 `if-else if` 문을 줄여 코드 실행 경로를 좀 더 효율적으로 만들 수 있습니다.

## 2. 공간 복잡도 개선 방법

**결론: 공간 복잡도는 `O(N*M)`이며, 이는 지도 저장에 필수적이므로 더 효율적인 방법은 없습니다.**

*   **현재 코드의 공간 복잡도:**
    *   `arr` (방 지도): `N*M` 크기의 2차원 배열로, `O(N*M)` 공간을 차지합니다.
    *   `dx`, `dy` (델타 배열): 상수 크기 `O(1)` 공간을 차지합니다.
    *   기타 변수: `N, M, r, c, d, ans, isContinue` 등은 모두 상수 `O(1)` 공간을 차지합니다.
*   따라서 전체 공간 복잡도는 `O(N*M)`입니다.
*   **개선 방법:** 문제 해결을 위해 방의 상태를 저장해야 하므로, `N*M` 크기의 배열은 필수적입니다. N과 M의 크기가 작기 때문에 (최대 50), 이 정도의 공간 사용은 문제가 되지 않습니다. 희소 행렬 같은 특별한 경우가 아니라면, 일반적인 2차원 배열이 가장 적합합니다.

## 3. 가독성 개선

가독성 개선을 위한 주요 전략은 다음과 같습니다.

*   **매직 넘버(Magic Number) 제거:** 방향을 나타내는 `0, 1, 2, 3`과 델타 배열 인덱스에 상수 변수 이름을 부여하여 의미를 명확하게 합니다.
*   **변수명 개선:** `cleaned`, `ans`, `isContinue` 등의 변수명을 더 직관적으로 변경합니다 (`foundUncleanedNeighbor`, `cleanedCount`, `robotOperating`).
*   **코드 중복 제거 (핵심 개선):**
    *   `if-else if` 문을 사용하여 후진 방향을 계산하는 부분을 `(d + 2) % 4`와 같이 일반화합니다.
    *   `if-else if` 문을 사용하여 좌회전 방향을 계산하는 부분을 `(d - 1 + 4) % 4`와 같이 일반화합니다. `+4`는 음수 결과 방지용입니다.
*   **델타 배열 이름 변경:** `dx`, `dy`를 `DR` (Delta Row), `DC` (Delta Column)으로 변경하여 행과 열을 명확히 합니다.
*   **`Scanner` 리소스 닫기:** `sc.close()`를 추가하여 리소스 누수를 방지합니다.

## 4. 최종 최적화된 코드

```java
package BOJ;

import java.util.Scanner;

public class BOJ_14503_Optimized {

    // 방향 상수 정의: 북(0), 동(1), 남(2), 서(3)
    private static final int NORTH = 0;
    private static final int EAST = 1;
    private static final int SOUTH = 2;
    private static final int WEST = 3;

    // 델타 배열: 각 방향으로 이동했을 때의 (dr, dc) 변화량
    // 순서: 북, 동, 남, 서 (d 값과 일치)
    private static final int[] DR = {-1, 0, 1, 0}; // delta row
    private static final int[] DC = {0, 1, 0, -1}; // delta column

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        // 방의 크기 N, M
        int N = sc.nextInt();
        int M = sc.nextInt();

        // 로봇 청소기 초기 좌표 (r, c) 및 방향 (d)
        int r = sc.nextInt();
        int c = sc.nextInt();
        int d = sc.nextInt(); // 0:북, 1:동, 2:남, 3:서

        // 방 생성 및 초기 상태 입력
        // 0: 청소되지 않은 칸, 1: 벽, 2: 청소된 칸
        int[][] room = new int[N][M];
        for (int i = 0; i < N; i++) {
            for (int j = 0; j < M; j++) {
                room[i][j] = sc.nextInt();
            }
        }
        
        sc.close(); // Scanner 리소스 닫기

        int cleanedCount = 0; // 청소한 칸의 개수
        boolean robotOperating = true; // 로봇 청소기가 작동 중인지 여부

        // 로봇 청소기 작동 시뮬레이션
        while (robotOperating) {
            // 1. 현재 칸이 아직 청소되지 않았다면 청소한다.
            if (room[r][c] == 0) {
                room[r][c] = 2; // 청소된 칸으로 표시
                cleanedCount++;
            }

            // 2. 현재 칸의 주변 4칸 중 청소되지 않은 빈 칸이 있는지 확인한다.
            boolean foundUncleanedNeighbor = false;
            for (int i = 0; i < 4; i++) {
                int nr = r + DR[i];
                int nc = c + DC[i];

                // 경계 조건 및 벽이 아닌지, 청소되지 않은 칸인지 확인
                if (nr >= 0 && nr < N && nc >= 0 && nc < M && room[nr][nc] == 0) {
                    foundUncleanedNeighbor = true;
                    break; // 하나라도 찾으면 더 이상 확인할 필요 없음
                }
            }

            // 주변 4칸 중 청소되지 않은 빈 칸이 없는 경우
            if (!foundUncleanedNeighbor) {
                // 2-1. 바라보는 방향을 유지한 채로 한 칸 후진할 수 있다면 한 칸 후진하고 1번으로 돌아간다.
                // 후진 방향은 현재 방향 d의 반대 방향: (d + 2) % 4
                int backDirection = (d + 2) % 4; 
                int backR = r + DR[backDirection];
                int backC = c + DC[backDirection];

                // 후진하려는 칸이 벽이라면 작동을 멈춘다.
                if (room[backR][backC] == 1) {
                    robotOperating = false;
                } else {
                    // 벽이 아니면 후진
                    r = backR;
                    c = backC;
                    // 후진 후 1번으로 돌아가 청소 여부를 다시 확인 (현재 위치는 청소되어 있을 수도, 아닐 수도 있음)
                }
            } 
            // 주변 4칸 중 청소되지 않은 빈 칸이 있는 경우
            else {
                // 2-2. 반시계 방향으로 90도 회전한다.
                // d: 0(북) -> 3(서), 1(동) -> 0(북), 2(남) -> 1(동), 3(서) -> 2(남)
                // (현재 방향 - 1 + 4) % 4 로 계산 가능
                d = (d - 1 + 4) % 4; 

                // 바라보는 방향을 기준으로 앞쪽 칸이 청소되지 않은 빈 칸인 경우 한 칸 전진한다.
                int frontR = r + DR[d];
                int frontC = c + DC[d];

                if (frontR >= 0 && frontR < N && frontC >= 0 && frontC < M && room[frontR][frontC] == 0) {
                    r = frontR;
                    c = frontC;
                    // 전진 후 1번으로 돌아가 청소 (현재 위치는 반드시 0이므로 청소될 것임)
                }
                // 앞쪽 칸이 빈 칸이 아니면(벽이거나 이미 청소된 칸) 전진하지 않고 1번으로 돌아간다.
                // (방향만 바꾼 채로 현재 위치에서 다시 주변 탐색을 시작함)
            }
        }

        System.out.println(cleanedCount);
    }
}
```

## 5. 왜 이 방법이 최선인지 설명

1.  **시간 복잡도 `O(N*M)` 유지 및 상수 계수 감소:**
    *   문제의 로봇 청소기 작동 방식은 본질적으로 시뮬레이션입니다. `N*M` 크기의 방에서 로봇은 최대 모든 칸을 한 번씩 청소할 수 있으며, 이 과정에서 주변 탐색, 방향 전환, 후진 등 상수 시간의 연산을 반복합니다. 따라서 `O(N*M)`보다 더 빠른 알고리즘을 설계하는 것은 불가능합니다.
    *   최적화된 코드는 `if-else if` 체인으로 인한 불필요한 조건 분기를 `(d + 2) % 4` 및 `(d - 1 + 4) % 4`와 같은 일반화된 수식으로 대체했습니다. 이는 코드의 분기 예측 실패(branch prediction miss) 가능성을 줄이고, 컴파일러가 더 효율적인 기계어를 생성하도록 도와 잠재적으로 실행 시간을 미세하게 단축시킵니다 (상수 시간 계수 감소).

2.  **공간 복잡도 `O(N*M)` 유지 (필수불가결):**
    *   방의 상태(청소 여부, 벽 여부)를 저장해야 하므로 `N*M` 크기의 2차원 배열은 필수적인 데이터 구조입니다. `N, M`이 최대 50이므로, `50*50 = 2500`개의 `int`를 저장하는 것은 메모리 측면에서 전혀 부담이 없습니다. 따라서 공간 복잡도 측면에서 더 나은 방법은 존재하지 않습니다.

3.  **가독성 및 유지보수성 대폭 개선:**
    *   **매직 넘버 제거:** `NORTH`, `EAST`, `SOUTH`, `WEST`와 같은 상수를 사용하여 `0, 1, 2, 3`이 어떤 방향을 의미하는지 즉각적으로 파악할 수 있습니다. 이는 코드의 의미를 명확히 하고 오류 발생 가능성을 줄입니다.
    *   **변수명 개선:** `ans` 대신 `cleanedCount`, `isContinue` 대신 `robotOperating` 등 의미를 더 잘 전달하는 변수명을 사용했습니다.
    *   **코드 중복 제거:** 가장 큰 개선점입니다. 후진 로직과 좌회전/전진 로직을 일반화된 수식으로 처리함으로써, 길고 반복적인 `if-else if` 블록을 제거했습니다. 이는 코드를 훨씬 간결하게 만들고, 특정 방향 처리 로직을 수정할 때 발생할 수 있는 오류를 방지합니다. 또한, 코드의 전체적인 흐름을 이해하기 쉽게 만듭니다.
    *   **`Scanner` 리소스 관리:** `sc.close()`를 추가하여 자원 누수를 방지하고 프로그램 안정성을 높였습니다.

이러한 개선 사항들은 문제의 근본적인 시간/공간 복잡도를 바꾸지는 않지만, 코드의 효율성 (상수 계수), 가독성, 유지보수성을 극대화하여 실용적인 관점에서 "최선"의 코드라고 할 수 있습니다.
