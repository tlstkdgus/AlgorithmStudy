# 코드 리뷰

**파일**: 이원준/202603/11/SWEA_2105.java

주신 코드는 SW Expert Academy의 디저트 카페 (SWEA 2105) 문제 풀이로 보입니다. DFS(깊이 우선 탐색)를 사용하여 주어진 격자에서 중복되지 않는 디저트 종류로 이루어진 가장 긴 사각형(마름모) 경로를 찾는 문제입니다.

아래는 코드에 대한 비판적이고 건설적인 리뷰입니다.

---

### 1. 시간/공간 복잡도

*   **시간 복잡도:**
    *   문제에서 `N`의 최대 값은 20입니다.
    *   **시작점 선택:** `N * N` (약 `N^2`)개의 모든 셀을 시작점으로 시도합니다. `for(int i = 0; i < N-1; i++)`와 `for(int j = 0; j < N-1; j++)`로 시작점이 `(0,0)`부터 `(N-2, N-2)`까지 탐색됩니다.
    *   **DFS 탐색:** `dfs` 함수는 재귀적으로 호출됩니다.
        *   각 DFS 상태는 `(r, c, dir)`로 정의될 수 있습니다. `r, c`는 `N^2`가지, `dir`는 4가지입니다. 따라서 `O(N^2 * 4)`개의 고유한 `(r, c, dir)` 상태가 있습니다.
        *   각 상태에서 최대 2개의 재귀 호출이 발생합니다 (`dir` 또는 `dir+1`).
        *   `visitedDessert` 배열을 통해 중복 디저트를 방문하는 경로를 가지치기합니다. 디저트 종류의 최대 개수는 100입니다.
        *   최대 경로 길이는 `4 * (N-1)`입니다 (예: `N=20`이면 최대 76개). 이 길이만큼 재귀 깊이가 깊어집니다.
        *   이러한 백트래킹(DFS) 문제의 정확한 시간 복잡도는 경로의 조합 수에 따라 매우 달라지지만, 일반적인 상한은 `O(N^2 * (N^2 * 4 * max_path_len))`과 같이 계산될 수 있습니다. 그러나 `visitedDessert`와 같은 가지치기가 있으므로 실제로는 `O(N^2 * 4 * N)` 또는 `O(N^3)` 정도로 근사할 수 있습니다.
    *   **총 시간 복잡도:** `N^2` (시작점) * `O(N^3)` (DFS) = `O(N^5)`로 추정됩니다.
        *   `N=20`일 때 `20^5 = 3,200,000`으로, 초당 1억 연산 기준으로 꽤 많은 연산이지만 충분히 제한 시간(보통 1초 또는 2초) 내에 통과할 수 있는 수준입니다. (물론 최악의 경우, 가지치기가 많이 안 일어난다면 더 걸릴 수도 있습니다.)

*   **공간 복잡도:**
    *   `map` 배열: `O(N^2)` (N*N 정수)
    *   `visitedDessert` 배열: `O(1)` (고정된 101 크기의 boolean 배열)
    *   재귀 스택: `O(N)` (DFS의 최대 깊이가 경로 길이와 같으므로)
    *   **총 공간 복잡도:** `O(N^2)` 입니다. `N=20`일 때 `20*20 = 400`으로 매우 효율적입니다.

### 2. 논리 오류

1.  **`visitedDessert` 초기화 위치 (매우 중요):**
    *   `visitedDessert = new boolean[101];` 이 한 번만 `for(int tc = 1; tc <= T; tc++)` 루프 안에서 초기화되고 있습니다.
    *   하지만 `maxDessert`를 찾기 위해 모든 `(startX, startY)` 쌍에 대해 DFS를 시작할 때마다 `visitedDessert` 배열은 *새롭게 초기화*되어야 합니다. 그렇지 않으면 이전 시작점에서 방문했던 디저트 종류가 다음 시작점에서의 경로 탐색에 영향을 미쳐 잘못된 결과(더 짧은 경로가 반환되거나 아예 경로를 찾지 못함)를 초래합니다.
    *   **수정 필요:** `visitedDessert = new boolean[101];` 코드를 `for(int j = 0; j < N-1; j++)` 루프 안에 넣어야 합니다.

2.  **`main` 루프의 시작점 범위 (비효율성 및 잠재적 논리 오류):**
    *   현재 `for(int i = 0; i < N-1; i++) { for(int j = 0; j < N-1; j++) { ... } }` 로 시작점을 탐색합니다. 즉, `i`는 `0`부터 `N-2`까지, `j`는 `0`부터 `N-2`까지 가능합니다.
    *   그러나 마름모 형태의 경로는 반드시 최소한 4개의 서로 다른 방향(SE, SW, NW, NE)으로 각 1칸 이상 이동해야 합니다. 즉, 각 변의 길이가 최소 1이어야 합니다.
    *   이를 고려하면 시작점 `(startX, startY)`는 다음 조건을 만족해야 합니다.
        *   `startX`는 `0`부터 `N-3`까지 (`startX + 1(SE) + 1(SW) < N` 을 만족해야 함)
        *   `startY`는 `1`부터 `N-2`까지 (`startY - 1(SW) >= 0` 및 `startY + 1(SE) < N` 을 만족해야 함)
    *   예를 들어, `startX = N-2`이거나 `startY = 0`인 경우, 유효한 마름모 경로를 만들 수 없습니다. 현재 코드는 이런 시작점에 대해서도 `dfs`를 호출하고, `dfs` 내부의 경계 조건 (`next_R < 0 || next_R >= N || next_C < 0 || next_C >= N`)에 의해 필터링되겠지만, 불필요한 탐색을 많이 수행하게 됩니다.
    *   **수정 필요:** `main` 루프의 범위를 `for(int i = 0; i < N-2; i++)` 및 `for(int j = 1; j < N-1; j++)`로 조정하는 것이 더 효율적이고 논리적으로 정확합니다. (`i`는 `0`부터 `N-3`까지, `j`는 `1`부터 `N-2`까지).

3.  **첫 번째 디저트의 `visitedDessert` 초기화 및 해제:**
    *   `visitedDessert[map[i][j]] = true;`
    *   `dfs(i, j, 0, 1);`
    *   `visitedDessert[map[i][j]] = false;`
    *   첫 번째 디저트 `map[i][j]`를 `true`로 설정하고 `dfs` 호출 후 `false`로 되돌리는 것은 올바른 백트래킹 패턴입니다. 특히 `visitedDessert`가 각 시작점마다 초기화되는 것으로 수정된다면, 이 `false`로 되돌리는 작업은 불필요하지만 해가 되지는 않습니다 (어차피 다음 시작점에서는 배열이 새로 생성되므로). 하지만 `dfs` 내부의 `true/false`는 필수적입니다.

### 3. 더 효율적인 방법

*   주어진 `N`의 범위(최대 20)를 고려할 때, 현재 DFS/백트래킹 방식은 일반적으로 허용되는 효율적인 접근 방식입니다. 특별히 더 복잡한 DP(동적 계획법)나 다른 알고리즘으로 전환할 필요는 없어 보입니다.
*   **미세한 최적화:** `dx`, `dy` 배열을 4방향으로 나누어 각각의 방향 전환 시점을 명시적으로 관리하는 방식도 있습니다. (예: `dfs(r, c, d1_len, d2_len, d3_len, d4_len)`).
    *   현재 코드의 `dfs(r, c, dir, count)`에서 `dir` 인자를 통해 `for(int i = dir; i < dir+2; i++)`로 방향 전환을 제어하는 방식은, 각 방향으로 일정 거리 이상 이동한 후 다음 방향으로 꺾는 형태를 잘 구현하고 있습니다. `dir=0`일 때는 `dir=0` 또는 `dir=1`로 이동할 수 있고, `dir=1`일 때는 `dir=1` 또는 `dir=2`로 이동할 수 있습니다. `dir=3`일 때는 `dir=3`만 이동할 수 있습니다 (`i=4`에서 `break`). 이 구현 자체는 논리적으로 올바릅니다.

### 4. 놓친 엣지 케이스

*   **`N < 3`인 경우:** 마름모는 최소 4개의 칸으로 이루어지므로, `N < 3`일 때는 유효한 마름모 경로를 만들 수 없습니다.
    *   현재 코드에서 `N=1` 또는 `N=2`인 경우, `main` 루프의 `i < N-1` 또는 `j < N-1` 조건에 따라 루프가 아예 실행되지 않거나, 실행되더라도 `dfs` 내에서 경계 조건에 걸려 `maxDessert`는 초기값인 `-1`을 유지하게 됩니다. 이는 올바른 처리입니다.

### 5. 코드 개선점

1.  **논리 오류 수정 (가장 중요):** 위에서 언급한 `visitedDessert` 초기화 위치와 `main` 루프의 시작점 범위를 수정합니다.

    ```java
    // 수정된 main 루프 예시
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        T = sc.nextInt();
        for(int tc = 1; tc <= T; tc++) {
            N = sc.nextInt();
            map = new int[N][N];
            for(int i = 0; i < N; i++) {
                for(int j = 0; j < N; j++) {
                    map[i][j] = sc.nextInt();
                }
            }
            
            maxDessert = -1;
            // i는 0부터 N-3까지 (최소 2칸 아래로 내려가야 함)
            // j는 1부터 N-2까지 (최소 1칸 왼쪽, 1칸 오른쪽으로 가야 함)
            for(int i = 0; i < N-2; i++) { // N-2는 N-3까지 포함
                for(int j = 1; j < N-1; j++) { // N-1은 N-2까지 포함
                    startX = i;
                    startY = j;
                    
                    visitedDessert = new boolean[101]; // 각 시작점마다 초기화!
                    visitedDessert[map[i][j]] = true; // 시작점 디저트 방문 처리
                    
                    dfs(i, j, 0, 1);
                    
                    // visitedDessert[map[i][j]] = false; // 선택사항: 이미 배열이 새로 초기화되므로 불필요하지만 해는 없음
                }
            }
            System.out.println("#"+tc+" "+maxDessert);
        }
    }
    ```

2.  **전역 변수 사용 최소화 (선택 사항):**
    *   `N`, `map`, `startX`, `startY`와 같은 변수들을 `dfs` 메서드의 인자로 전달하는 것도 코드의 가독성과 모듈성을 높이는 방법입니다. 그러나 경쟁 프로그래밍에서는 전역 변수 사용이 일반적이므로 필수는 아닙니다.
    *   `visitedDessert` 배열은 각 `dfs` 호출에서 상태를 저장하고 복원해야 하므로 전역으로 두되, 각 시작점마다 `new boolean[101]`로 초기화하는 것이 중요합니다.

3.  **`dfs` 함수의 `dir` 인자 주석:**
    *   `dir` 인자가 현재 탐색 가능한 최소 방향 인덱스를 나타내므로, 해당 동작에 대한 주석을 추가하면 코드 이해에 도움이 될 것입니다.

---

전반적으로, `visitedDessert` 초기화 오류를 제외하면 디저트 카페 문제를 해결하기 위한 견고한 DFS 백트래킹 로직을 잘 구현하셨습니다. `N`이 20인 환경에서 `O(N^5)` 복잡도는 적절한 풀이입니다. 위에 언급된 두 가지 주요 수정사항을 적용하면 완벽한 코드가 될 것입니다.
