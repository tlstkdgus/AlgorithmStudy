# 최적화 분석

제공해주신 코드는 삼성 SW 역량 테스트 (SWEA) 1952. 수영장 문제의 전형적인 완전 탐색 (재귀 호출 + 백트래킹) 풀이입니다. 이 문제의 핵심은 특정 달까지의 최소 비용을 계산하는 것으로, 동적 계획법 (Dynamic Programming, DP)을 사용하여 효율적으로 최적화할 수 있습니다.

---

### 1. 시간 복잡도 개선 방법

**문제점:**
현재 재귀 함수 `swimfee(int m, int fee)`는 각 월 `m`마다 세 가지 선택지 (1일권, 1달권, 3달권)를 모두 탐색합니다. 이는 `m`이 1부터 12까지 변하므로, 최악의 경우 $3^{12}$에 가까운 조합을 탐색하게 됩니다. `fee >= ans`라는 가지치기(pruning)가 있지만, 근본적인 중복 계산을 막지는 못합니다. 동일한 `m`에 대해 `fee`만 다른 수많은 호출이 발생하며, 이들 중 많은 경우가 불필요하게 계산됩니다.

**개선 방법:**
**동적 계획법 (Dynamic Programming, DP)**을 사용합니다.
`dp[i]`를 "i월까지의 최소 이용 요금"이라고 정의합니다. 각 월 `i`에 대해 `dp[i]`를 계산할 때, 이전에 계산된 `dp[j]` (j < i) 값들을 활용하여 중복 계산을 피할 수 있습니다.

**DP 점화식:**
1.  **초기값:** `dp[0] = 0` (0월까지의 비용은 0).
2.  **점화식:** 각 월 `i` (1부터 12까지)에 대해 `dp[i]`를 다음과 같이 계산합니다.
    *   **옵션 1: 1일권 또는 1달권으로 `i`월을 해결하는 경우**
        `dp[i] = dp[i-1] + Math.min(month[i] * d, m1);`
        (이전 달까지의 최소 비용 `dp[i-1]`에 이번 달 `i`에 대한 최소 비용 (1일권 x 사용일수 vs 1달권)을 더합니다. `month[i]`가 0일 경우, `0 * d`는 0이므로 `Math.min(0, m1)`은 0이 되어 이 달에는 추가 비용이 발생하지 않음을 자연스럽게 처리합니다.)
    *   **옵션 2: 3달권으로 `i`월을 포함하는 경우**
        `i`월이 3달권의 마지막 달이라고 가정하면, 이 3달권은 `i-2`, `i-1`, `i`월을 커버합니다. 따라서 `i-3`월까지의 최소 비용 `dp[i-3]`에 3달권 비용 `m3`를 더한 값과 비교합니다. 이 옵션은 `i`가 최소 3 이상일 때만 가능합니다.
        `if (i >= 3) { dp[i] = Math.min(dp[i], dp[i-3] + m3); }`
3.  **최종 결과:** 12월까지 계산된 `dp[12]` 값과 1년권 `y` 중 더 작은 값이 최종 답이 됩니다.
    `ans = Math.min(dp[12], y);`

**시간 복잡도 변화:**
*   **기존:** $O(3^{12})$ (최악의 경우, 가지치기로 실제는 줄어들지만 여전히 지수적)
*   **개선 후:** $O(12)$ (단순 반복문으로 12번의 계산만 수행) = $O(1)$ (상수 시간)
    이는 12개월이라는 고정된 작은 N 값에 대해 압도적인 성능 향상입니다.

### 2. 공간 복잡도 개선 방법

**문제점:**
현재 코드는 재귀 호출 스택을 사용합니다. 최대 12단계 깊이의 스택이 쌓일 수 있습니다. `month` 배열도 13개의 int 값을 저장합니다.

**개선 방법:**
DP 배열 `dp`를 사용합니다. `dp` 배열은 13개의 int 값을 저장합니다. `month` 배열도 13개의 int 값을 저장합니다. 이는 기존의 재귀 스택 공간과 유사하거나 오히려 더 적은 양의 메모리를 사용합니다.

**공간 복잡도 변화:**
*   **기존:** $O(N)$ (재귀 스택 깊이, N=12) + $O(N)$ (`month` 배열) = $O(1)$ (N이 상수이므로)
*   **개선 후:** $O(N)$ (`dp` 배열) + $O(N)$ (`month` 배열) = $O(1)$ (N이 상수이므로)
    공간 복잡도 면에서는 큰 차이가 없으며, 둘 다 매우 효율적입니다. 하지만 DP 방식은 재귀 함수 호출 오버헤드가 없다는 점에서 미미하게 더 좋습니다.

### 3. 가독성 개선

**개선 방법:**
*   **변수명 명확화:** `d`, `m1`, `m3`, `y`와 같은 짧은 변수명 대신 `dailyTicketCost`, `monthlyTicketCost`, `quarterlyTicketCost`, `yearlyTicketCost` 와 같이 의미를 명확하게 드러내는 이름을 사용합니다.
*   **주석 추가:** DP 배열의 정의, 점화식의 각 단계, 그리고 특별한 처리 (예: 3달권 적용 조건) 등에 대한 주석을 추가하여 코드의 흐름과 로직을 쉽게 이해할 수 있도록 합니다.
*   **함수 분리 제거:** 이 문제의 DP 풀이는 `main` 메서드 내에서 깔끔하게 구현 가능하므로, 별도의 재귀 함수 `swimfee`를 제거합니다.

### 4. 최종 최적화된 코드

```java
    package com.ssafy.swea;

    import java.util.Scanner;
    import java.lang.Math; // Math.min 사용을 위해 명시적으로 임포트

    public class SWEA_1952_Optimized {
        public static void main(String[] args) {
            Scanner sc = new Scanner(System.in);
            
            int T = sc.nextInt();
            
            for(int tc = 1; tc <= T; tc++) {
                // 이용권 가격 변수명 변경: 의미를 명확히 함
                int dailyTicketCost = sc.nextInt();    // 1일 이용권 가격
                int monthlyTicketCost = sc.nextInt();  // 1달 이용권 가격
                int quarterlyTicketCost = sc.nextInt(); // 3달 이용권 가격
                int yearlyTicketCost = sc.nextInt();   // 1년 이용권 가격
                
                int[] monthUsageDays = new int[13]; // 1월부터 12월까지의 이용 일수 저장
                for(int i = 1; i <= 12; i++) {
                    monthUsageDays[i] = sc.nextInt();
                }
                
                // 동적 계획법 (Dynamic Programming)을 위한 dp 배열
                // dp[i] : i월까지의 최소 이용 요금
                int[] dp = new int[13];
                
                // dp[0]은 0으로 초기화. 0월까지는 이용 요금이 없으므로.
                dp[0] = 0; 
                
                // 1월부터 12월까지 순차적으로 최소 비용 계산
                for(int i = 1; i <= 12; i++) {
                    // 1. 전 달까지의 요금 (dp[i-1])에 이번 달(i) 요금을 더하는 방식
                    //    이번 달 요금은 (일일권 x 사용일수)와 (1달권) 중 더 저렴한 것으로 선택
                    //    만약 monthUsageDays[i]가 0이면, monthUsageDays[i] * dailyTicketCost는 0이 되어, 
                    //    해당 월에는 비용이 추가되지 않도록 자연스럽게 처리됨 (Math.min(0, monthlyTicketCost) = 0).
                    dp[i] = dp[i-1] + Math.min(monthUsageDays[i] * dailyTicketCost, monthlyTicketCost);
                    
                    // 2. 3달권을 사용하는 경우 (이번 달 i가 3달권의 마지막 달인 경우)
                    //    dp[i-3] (i-3월까지의 요금)에 3달권 요금을 더한 값과 현재 dp[i]를 비교하여 더 작은 값으로 업데이트
                    //    단, i가 최소 3 이상이어야 3달권을 고려할 수 있음 (i-2, i-1, i 세 달 커버)
                    if (i >= 3) {
                        dp[i] = Math.min(dp[i], dp[i-3] + quarterlyTicketCost);
                    }
                }
                
                // 최종 결과는 12월까지의 최소 요금 (dp[12])과 1년 이용권 요금 (yearlyTicketCost) 중 더 저렴한 것
                int minTotalCost = Math.min(dp[12], yearlyTicketCost);
                
                System.out.println("#" + tc + " " + minTotalCost);
            }
            sc.close(); // Scanner 리소스 해제
        }
    }
```

### 5. 왜 이 방법이 최선인지 설명

이 동적 계획법 (DP) 방식이 최선인 이유는 다음과 같습니다:

1.  **압도적인 시간 복잡도 개선:**
    *   기존의 재귀 호출 방식은 각 달마다 여러 선택지를 탐색하며 중복 계산이 발생하여 시간 복잡도가 지수적($O(3^{N})$)으로 증가합니다. 이는 N(개월 수, 12)이 조금만 커져도 계산 불가능한 수준이 됩니다.
    *   DP 방식은 각 월 `i`에 대해 `dp[i]` 값을 한 번만 계산하고, 이전에 계산된 `dp[j]` 값을 재활용합니다. 따라서 1월부터 12월까지 단 한 번의 반복문으로 모든 계산을 완료하며, 시간 복잡도는 $O(N)$ (여기서는 $O(12) = O(1)$)으로 상수 시간에 가까운 성능을 보장합니다. 이는 이 문제에서 달의 수가 12로 고정되어 있기 때문에 가능한 극도로 효율적인 접근입니다.

2.  **명확하고 간결한 로직:**
    *   `dp[i]`의 정의("i월까지의 최소 비용")가 명확하여, 각 월의 선택지를 점화식으로 간결하게 표현할 수 있습니다.
    *   재귀 함수 호출 스택 관리, 가지치기 조건 등을 복잡하게 고려할 필요 없이, 순차적인 반복문으로 문제를 해결할 수 있어 코드의 이해와 디버깅이 더 용이합니다.

3.  **최소한의 공간 사용:**
    *   `dp` 배열의 크기는 월의 수(13)에 비례하며, 이는 입력 크기에 대해 상수($O(1)$)에 해당하는 매우 작은 메모리 사용량입니다. 이는 기존의 재귀 스택 메모리 사용량과 비슷하거나 더 효율적입니다.

4.  **최적성 보장:**
    *   동적 계획법은 "최적 부분 구조"와 "중복 부분 문제"라는 두 가지 속성을 만족하는 문제에 대해 최적해를 찾는 가장 효율적인 방법 중 하나입니다. 이 문제는 정확히 이 두 가지 속성을 모두 가지므로, DP를 통해 도출된 해는 항상 최적해임이 보장됩니다.

결론적으로, N이 작은 고정된 값일 때 DP는 종종 완전 탐색이나 재귀 + 메모이제이션보다 더욱 직관적이고 성능이 우수한 해결책을 제공합니다. 특히 이 문제처럼 각 단계의 결정이 이전 단계의 최적 결과에만 의존하고 미래의 결정에 직접적인 영향을 미치지 않는 경우 (즉, "무기억 속성"을 가질 때), DP가 최적의 선택입니다.
