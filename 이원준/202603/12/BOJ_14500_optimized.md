# 최적화 분석

주어진 코드는 백준 14500번 (테트로미노) 문제를 해결하기 위한 브루트포스(DFS) 방식입니다. 현재 코드의 문제점은 'T' 모양 테트로미노를 찾기 위해 `if(depth == 2)`와 같은 특이한 로직을 DFS 내부에 포함하고 있다는 점입니다. 이 방식은 이해하기 어렵고, 다른 형태의 테트로미노를 찾는 데는 불필요하며, 특정 'T' 모양의 회전 및 대칭을 모두 찾는다는 보장도 명확하지 않습니다.

최적화는 이 부분을 개선하고 전반적인 가독성을 높이는 데 중점을 둡니다.

---

### 분석 및 최적화 방향

BOJ 14500번 문제의 핵심은 5가지 종류의 테트로미노(I, O, T, J/L, S/Z)가 만드는 최대 합을 찾는 것입니다. 4개의 칸으로 이루어진 테트로미노는 크게 두 가지 방식으로 분류할 수 있습니다:
1.  **경로 기반 테트로미노:** 4개의 칸이 모두 인접하여 하나의 '경로'를 이루는 형태. (I, O, J/L, S/Z)
2.  **'T' 모양 테트로미노:** 중심 칸 하나에 3개의 칸이 인접한 형태. 이 형태는 단순한 4칸 경로로만 찾기 어렵습니다.

현재 코드는 DFS를 통해 4칸 경로를 탐색하며, `depth == 2` 조건으로 'T' 모양을 어설프게 탐색하려 합니다.

**최적화 전략:**
1.  **DFS 재정의:** `visited` 배열을 이용한 일반적인 DFS를 통해 4칸 경로 형태의 테트로미노 (I, O, J/L, S/Z)를 모두 찾습니다. 이는 시작점과 방향에 따라 모든 회전/대칭 형태를 커버합니다.
2.  **'T' 모양 개별 처리:** 'T' 모양 테트로미노는 중심 칸 하나와 그 주변 3칸으로 이루어져 있습니다. DFS로 찾기 모호하므로, 모든 셀을 순회하며 해당 셀을 중심으로 가능한 4가지 'T' 모양 (4방향)을 직접 탐색하여 합을 계산합니다.

이 방법은 각 테트로미노 형태에 가장 적합한 방식으로 탐색을 분리하여 코드의 명확성과 정확성을 높입니다.

---

### 1. 시간 복잡도 개선 방법

*   **현재 코드의 DFS 시간 복잡도:**
    `N*M`개의 시작점에서 DFS를 시작하고, 각 DFS는 최대 깊이 4까지 탐색합니다. 각 단계에서 최대 4방향으로 분기하므로, 최악의 경우 `O(N*M * 4^4)`에 가까울 수 있습니다. 그러나 `visited` 배열 덕분에 실제로는 같은 경로를 반복하지 않아 훨씬 빠릅니다. 그래도 `if(depth == 2)` 조건이 불필요한 재탐색을 유발하거나, 특정 'T' 모양을 놓칠 가능성이 있습니다.

*   **개선된 코드의 시간 복잡도:**
    1.  **DFS (경로 기반 테트로미노):**
        `N*M`개의 시작점에서 DFS를 시작합니다. 각 DFS는 4단계 깊이까지 탐색하며, 방문했던 곳을 다시 가지 않으므로 실제 탐색 경로는 최대 `4 * 3 * 3 * 3` 정도입니다. 즉, 시작점마다 상수 시간(`O(1)`)에 가까운 연산을 수행합니다. 총 `O(N*M)`이 됩니다.
    2.  **'T' 모양 개별 처리:**
        `N*M`개의 모든 셀을 순회하며 각 셀에 대해 4가지 'T' 모양의 합을 계산합니다. 각 'T' 모양은 4칸으로 이루어져 있으므로 상수 시간(`O(1)`) 연산입니다. 총 `O(N*M)`이 됩니다.
    *   **총 시간 복잡도:** `O(N*M)`. 모든 셀을 한 번씩 시작점/중심점으로 고려해야 하므로 이보다 더 낮아질 수 없습니다. 이는 최적의 시간 복잡도입니다.

### 2. 공간 복잡도 개선 방법

*   **현재 코드의 공간 복잡도:**
    *   `arr`: `O(N*M)` (입력 배열)
    *   `visited`: `O(N*M)` (방문 여부 배열)
    *   재귀 스택: DFS 깊이가 최대 4이므로 `O(4)` 즉, `O(1)`
    *   총 `O(N*M)`

*   **개선된 코드의 공간 복잡도:**
    *   `arr`: `O(N*M)`
    *   `visited`: `O(N*M)`
    *   재귀 스택: DFS 깊이가 최대 4이므로 `O(4)` 즉, `O(1)`
    *   총 `O(N*M)`. 공간 복잡도는 입력 배열과 방문 배열이 필수적이므로 더 개선될 여지가 없습니다.

### 3. 가독성 개선

*   **`dfs` 함수의 명확성:** `depth == 2`와 같은 특수 로직을 제거하여, `dfs` 함수가 오직 '4칸 경로 탐색'이라는 목적에만 충실하게 만듭니다.
*   **함수 분리:** 'T' 모양 테트로미노를 탐색하는 로직을 `checkT`와 같은 별도의 함수로 분리하여 각 부분이 어떤 역할을 하는지 명확하게 합니다.
*   **상수 사용:** `dx`, `dy`와 같은 이동 배열을 `final`로 선언하여 상수로 사용함을 명확히 합니다.
*   **변수명:** `depth` 대신 `count`와 같이 현재까지 방문한 칸의 수를 나타내는 더 직관적인 변수명을 사용할 수 있습니다.

### 4. 최종 최적화된 코드

```java
package BruteForce;

import java.util.Scanner;

public class BOJ_14500_Optimized {
    static int N, M;
    static int[][] arr;
    static int maxVal;
    static boolean[][] visited; // DFS용 방문 배열

    // 상, 하, 좌, 우 이동
    static final int[] dx = {-1, 1, 0, 0};
    static final int[] dy = {0, 0, -1, 1};

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        N = sc.nextInt();
        M = sc.nextInt();

        arr = new int[N][M];
        for (int i = 0; i < N; i++) {
            for (int j = 0; j < M; j++) {
                arr[i][j] = sc.nextInt();
            }
        }
        sc.close(); // Scanner 자원 해제

        maxVal = 0;
        visited = new boolean[N][M];

        // 모든 셀을 시작점으로 하여 DFS 및 T-모양 탐색
        for (int i = 0; i < N; i++) {
            for (int j = 0; j < M; j++) {
                // 1. DFS를 통해 4칸 경로 테트로미노 (I, O, J/L, S/Z) 탐색
                visited[i][j] = true; // 현재 칸 방문 처리
                dfs(i, j, 1, arr[i][j]); // DFS 시작 (현재 칸, 방문 칸 수 1, 현재까지의 합)
                visited[i][j] = false; // 백트래킹 (현재 칸 방문 해제)

                // 2. T-모양 테트로미노는 별도로 탐색
                checkTShape(i, j);
            }
        }

        System.out.println(maxVal);
    }

    // DFS로 4칸 경로 테트로미노 (I, O, J/L, S/Z) 탐색
    static void dfs(int r, int c, int count, int sum) {
        if (count == 4) { // 4개의 칸을 모두 선택했다면
            maxVal = Math.max(maxVal, sum); // 최대값 갱신
            return;
        }

        for (int d = 0; d < 4; d++) { // 4방향 탐색
            int nr = r + dx[d];
            int nc = c + dy[d];

            // 경계 조건 및 방문 여부 확인
            if (nr >= 0 && nr < N && nc >= 0 && nc < M && !visited[nr][nc]) {
                visited[nr][nc] = true; // 다음 칸 방문 처리
                dfs(nr, nc, count + 1, sum + arr[nr][nc]); // 재귀 호출
                visited[nr][nc] = false; // 백트래킹 (다음 칸 방문 해제)
            }
        }
    }

    // T-모양 테트로미노 탐색 (4가지 회전/대칭 형태)
    // (r, c)를 T-모양의 한 부분으로 간주하여 고정된 형태로 탐색
    static void checkTShape(int r, int c) {
        // T-모양의 4가지 회전/대칭 형태의 상대 좌표
        // 첫 번째 좌표는 (r,c) 자신을 포함하는 것이 일반적
        // {{0,0}, {0,1}, {0,2}, {1,1}} 형태의 'T'는 첫 좌표가 (r,c)이고, (r,c+1), (r,c+2), (r+1,c+1) 같은 방식
        // 여기서는 (r,c)를 '중심' 칸으로 보고 나머지 3칸을 붙이는 방식으로 구현
        // 즉, 3칸이 (r,c)에 인접한 형태
        int[][] tShapes = {
            // T-모양: 중심(r,c) + 3방향
            {r, c,   r-1, c,   r+1, c,   r, c+1}, // 위, 아래, 오른쪽에 붙는 T
            {r, c,   r-1, c,   r+1, c,   r, c-1}, // 위, 아래, 왼쪽에 붙는 T
            {r, c,   r, c-1,   r, c+1,   r+1, c}, // 왼쪽, 오른쪽, 아래에 붙는 T
            {r, c,   r, c-1,   r, c+1,   r-1, c}  // 왼쪽, 오른쪽, 위에 붙는 T
        };

        for (int i = 0; i < tShapes.length; i++) {
            int currentSum = 0;
            boolean possible = true;
            for (int j = 0; j < 4; j++) { // 4칸을 순회
                int nr = tShapes[i][j * 2];     // 행 좌표
                int nc = tShapes[i][j * 2 + 1]; // 열 좌표

                // 경계 조건 확인
                if (nr < 0 || nr >= N || nc < 0 || nc >= M) {
                    possible = false; // 하나라도 벗어나면 불가능
                    break;
                }
                currentSum += arr[nr][nc];
            }

            if (possible) {
                maxVal = Math.max(maxVal, currentSum); // 가능한 T-모양이라면 최대값 갱신
            }
        }
    }
}

```

### 5. 왜 이 방법이 최선인지 설명

1.  **정확성 보장:**
    *   **DFS를 통한 경로 기반 테트로미노 탐색:** I, O, J, L, S, Z 형태의 테트로미노는 4개의 칸이 서로 인접한 '경로'를 이룹니다. DFS는 이러한 모든 4칸 경로를 효율적으로 탐색합니다. `visited` 배열을 이용한 백트래킹 덕분에 중복 계산 없이 정확하게 모든 경우를 탐색합니다. 각 셀을 시작점으로 하여 DFS를 시작하므로 모든 가능한 회전 및 대칭 형태가 자연스럽게 커버됩니다.
    *   **'T' 모양 테트로미노 개별 처리:** 'T' 모양은 3개의 칸이 한 칸에 붙어 있는 형태이며, 이는 DFS의 단순한 4칸 경로 탐색으로는 놓치거나 복잡한 예외 처리가 필요합니다. 이 방법은 모든 셀을 'T' 모양의 중심 또는 기준점으로 삼아 4가지 고정된 'T' 형태를 직접 확인하므로, 모든 'T' 모양의 회전 및 대칭 형태를 빠짐없이 탐색합니다.

2.  **효율성 (시간 복잡도 `O(N*M)`):**
    *   각 셀을 시작점으로 DFS를 수행할 때, DFS 깊이가 4로 매우 짧습니다. 따라서 각 시작점에서의 DFS 연산은 상수 시간에 수렴합니다 (4차원 미로에서 4칸 탐색). 총 `N*M`개의 시작점이 있으므로 `O(N*M)`입니다.
    *   'T' 모양 테트로미노를 탐색하는 `checkTShape` 함수 역시 각 셀에 대해 4가지 고정된 형태를 확인하는 상수 시간 연산이므로 `O(N*M)`입니다.
    *   두 연산을 합쳐도 전체 시간 복잡도는 `O(N*M)`으로, 모든 셀을 적어도 한 번은 확인해야 하는 문제의 특성상 이론적인 최소 시간 복잡도에 도달합니다.

3.  **가독성 및 유지보수성:**
    *   `dfs` 함수는 표준적인 깊이 우선 탐색 로직만을 가집니다. 특정 케이스(`depth == 2`)에 대한 예외 처리가 없어 함수의 목적이 매우 명확합니다.
    *   `checkTShape` 함수는 'T' 모양 테트로미노만을 처리합니다. 이처럼 각기 다른 특성을 가진 테트로미노 탐색 로직을 분리함으로써 코드를 이해하고 디버깅하기 훨씬 용이해집니다.
    *   변수명이나 상수의 사용이 명확하여 코드의 의도를 쉽게 파악할 수 있습니다.

이 방법은 테트로미노 문제에서 'T' 모양의 특수성을 고려하면서 다른 모양들을 일반적인 방식으로 효율적으로 처리하는 표준적이고 가장 좋은 접근 방식입니다.
